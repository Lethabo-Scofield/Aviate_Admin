Build an upgraded logistics SaaS web application called Aviate Dispatch System.

We already have a working route optimization engine. Your task is to improve the system by making it explainable, modular, and visually clear, while enhancing the UI/UX for a production-ready SaaS dashboard.

1. UI / Design System

Use the following design rules:

Primary color: #008080 (teal)
Secondary color: navy blue accents
Background: pure white (#FFFFFF)
Style: clean, modern SaaS dashboard (like Stripe / Uber Freight)
Use cards, soft shadows, and clear spacing
Minimalistic but data-rich layout
Use icons where appropriate for clarity
2. Core Goal of Upgrade

Transform the route optimization output from a “black box result” into an explainable AI-style decision system.

Every route must clearly show:

Why a driver was selected
Why others were rejected
Full cost breakdown
Efficiency score
Alternative suggestions (if available)
3. System Architecture (must implement or simulate)

Break optimization into 4 layers:

Layer 1: Candidate Filtering
Filter drivers by:
distance to pickup
vehicle type match
availability
Layer 2: Scoring Engine

Each driver-job pair must generate a score:

score =
(distance_to_pickup × w1)

(extra_route_time × w2)
(idle_time × w3)
(lateness_risk × w4)

Lower score = better match

Layer 3: Route Sequencing
Optimize stop order per driver
Respect constraints:
pickup before drop-off
capacity limits
time windows
Layer 4: Real-time Adjustment
Detect changes:
traffic
cancellations
new jobs
Recalculate only affected routes
4. New Feature: Route Explainability Panel

For every assigned job, create a UI panel:

“Why this driver was chosen”

Show:

distance advantage
route efficiency gain
workload comparison
final score
“Rejected drivers”

List top 3 alternatives with:

reason they were not selected
score difference
5. New Feature: Route Breakdown Card

Each route must display:

Total distance
Total estimated time
Idle driving time
Cost estimate
Efficiency score (0–100)

Add a visual progress bar for efficiency.
